# คู่มือสำหรับผู้ใช้วิทยุ — เซิร์ฟเวอร์ ADN DMR

> **เอกสารนี้มีไว้สำหรับ:** ทุกคนที่ใช้วิทยุในการพูดและฟัง
> คุณไม่จำเป็นต้องรู้เรื่องเซิร์ฟเวอร์ เครือข่าย หรือการเขียนโปรแกรมใด ๆ

## แนวคิดพื้นฐาน (สิ่งที่คุณควรรู้ขั้นต่ำ)

### Talk Group (TG)

ตัวเลขที่ใช้ระบุช่อง ตัวอย่าง:
- **TG 730** — ห้องทั่วไปสำหรับประเทศชิลี
- **TG 714** — ห้องทั่วไปสำหรับประเทศปานามา
- **TG 4000** — หมายเลขพิเศษสำหรับ “วางสาย” (ดูด้านล่าง)

### Timeslot (slot)

DMR แบ่งความถี่แต่ละความถี่ออกเป็น **สองช่องเวลา**: **slot 1** และ **slot 2**
เปรียบเสมือนการมีเลนสองเลนบนทางด่วน: การสนทนาสองครั้งสามารถดำเนินไปพร้อมกันได้
โดยไม่รบกวนกัน hotspot และเซิร์ฟเวอร์ของคุณจะประสานงานนี้ให้

ในฐานะผู้ใช้ คุณแทบไม่ต้องคิดเรื่อง slot เลย: hotspot ของคุณได้รับการตั้งค่า
ให้ใช้ slot ที่ถูกต้องโดยอัตโนมัติ

### PTT (Push To Talk)

ปุ่มบนวิทยุของคุณ กดเพื่อพูด ปล่อยเพื่อฟัง

## สิ่งที่คุณทำได้

### คุยบน Talk Group

1. **เลือก TG** บนวิทยุของคุณ (เหมือนการเปลี่ยนช่อง)
2. **กด PTT** แล้วพูด
3. **ปล่อย PTT** เพื่อฟังการตอบกลับ

เท่านี้เอง เซิร์ฟเวอร์จะจัดการส่วนที่เหลือ

### เปิดใช้งาน Talk Group ใหม่ (แบบไดนามิก)

หากคุณต้องการฟัง TG ที่ **ไม่ได้ตั้งค่าไว้** บน hotspot ของคุณ
เพียงแค่ **ส่งสัญญาณบน TG นั้น** ตัวอย่าง: คุณไม่เคยใช้ TG 730507 แต่คุณ
ต้องการฟัง:

1. ตั้งโปรแกรม TG 730507 ลงในวิทยุของคุณ
2. กด PTT ค้างไว้หนึ่งวินาที (เพื่อ “แจ้ง” เซิร์ฟเวอร์)
3. จากช่วงเวลานั้นเป็นต้นไป **hotspot ของคุณจะเริ่มฟัง TG นั้น** จนกว่าคุณจะ
   ปิดการใช้งาน (โดยกด 4000) หรือหมดเวลา

> **สำคัญ:** บนเซิร์ฟเวอร์นี้ **การส่งสัญญาณเพียงครั้งเดียวก็เพียงพอ** ในการเปิดใช้งาน
> และฟัง คุณไม่ต้องส่งสองครั้ง

### ปิดการใช้งาน Talk Group (วางสาย)

หากคุณไม่ต้องการฟัง TG ที่คุณเปิดใช้งานอีกต่อไป **กด TG 4000** และกด PTT
เปรียบเสมือนการ “วางสายโทรศัพท์”:

1. เลือก **TG 4000** บนวิทยุของคุณ
2. กด PTT ค้างไว้หนึ่งวินาที
3. TG ไดนามิกจะถูกล้าง คุณจะไม่ได้ยินอีกต่อไป

> TG 4000 **จะล้างเฉพาะ TG ที่คุณเปิดใช้งานเท่านั้น** ไม่มีผลกระทบต่อ TG แบบสแตติก
> (ที่คุณตั้งค่าไว้ในบรรทัด OPTIONS ในแผงบริการตนเอง หรือบนเซิร์ฟเวอร์)

### ฟังเสียงของตัวเอง (echo)

อยากทดสอบว่าเสียงของคุณส่งถึงเซิร์ฟเวอร์อย่างถูกต้องหรือไม่? **กด TG 9990**
และกด PTT เซิร์ฟเวอร์จะเล่นเสียงที่บันทึกของคุณกลับมาเพื่อให้คุณตรวจสอบ
คุณภาพ

> echo จะกลับมา **เฉพาะ hotspot ของคุณ** แม้คุณจะลงทะเบียนหลายเครื่อง
> ด้วย DMR ID เดียวกันก็ตาม

### ฟังข้อความให้ข้อมูล

โดยการกด **9991 ถึง 9999** คุณจะได้ยินคลิปที่บันทึกไว้ล่วงหน้าซึ่งมีข้อมูล
บริการ (ขึ้นอยู่กับว่าผู้ดูแลระบบตั้งค่าคลิปใดไว้บ้าง)

## กฎที่เซิร์ฟเวอร์ใช้เพื่อคุณ (คุณไม่ต้องทำอะไรเลย)

เซิร์ฟเวอร์มีกฎอัตโนมัติเพื่อไม่ให้การสนทนาทับซ้อนกัน คุณเพียงแค่
พูด; เซิร์ฟเวอร์จะดูแลให้ทุกอย่างทำงาน

### หนึ่งการสนทนาต่อ TG ในแต่ละช่วงเวลา

หากมีคนกำลังคุยบน TG 730 **ไม่มีใครอื่นสามารถแทรกได้** บน TG
เดียวกันนั้น หากคุณกด PTT บน 730 ขณะที่คนอื่นกำลังพูด เซิร์ฟเวอร์จะ
ไม่สนใจคุณ (ไม่รบกวนผู้พูด) แต่ยังคงให้คุณฟังสิ่งที่พวกเขาพูด

### ไม่ตัดการเชื่อมต่อคุณหากมีคนเข้าร่วมช้า

หากคุณกำลังพูดและมีสมัครเล่นท่านอื่น **เชื่อมต่อในระหว่างทาง** เซิร์ฟเวอร์จะ
ส่งเสียงของคุณตั้งแต่ช่วงเวลานั้นเป็นต้นไป พวกเขาไม่ต้องรอให้คุณ
พูดจบก่อนจึงจะเริ่มได้ยินคุณ

### เวลารอระหว่าง TG (hangtime)

เมื่อการสนทนาสิ้นสุดลงบน slot หนึ่ง จะมีพัก **5 วินาที** ก่อนที่
slot นั้นจะรับ TG อื่น สิ่งนี้ป้องกันไม่ให้การสนทนาสองครั้งสับไขว้กัน
เป็นไปโดยอัตโนมัติ; คุณไม่สังเกตเห็น

### คุณไม่สามารถพูดและฟังพร้อมกันบน slot เดียวกันได้

หากคุณกำลังส่งสัญญาณบน slot 2 คุณไม่สามารถรับสายอื่นบน slot 2 ได้
จนกว่าคุณจะปล่อย PTT นี่เป็นเรื่องปกติ: วิทยุของคุณเองก็ทำสองอย่างพร้อมกัน
บน slot เดียวกันไม่ได้เช่นกัน

แต่ **คุณสามารถพูดบน slot 1 และฟังบน slot 2** พร้อมกันได้ (หาก
วิทยุของคุณเป็นแบบ full-duplex)

## การตั้งค่า hotspot ของคุณ

hotspot ของคุณ (Pi-Star, MMDVM ฯลฯ) ถูกตั้งค่าด้วยบรรทัดที่เรียกว่า
**OPTIONS** ซึ่งระบุว่าคุณต้องการฟัง TG ใด คุณไม่ต้องแก้ไขด้วยมือ
เอง: **แผงเว็บ** (บริการตนเอง) ให้คุณเปลี่ยน TG ของคุณจาก
เบราว์เซอร์ได้

### บรรทัด OPTIONS และรหัสผ่านของคุณ

hotspot ของคุณส่งบรรทัดที่เรียกว่า **OPTIONS** ไปยังเซิร์ฟเวอร์หลังจากเชื่อมต่อ เซิร์ฟเวอร์
จะรอ 10 วินาทีให้บรรทัดนั้นมาถึง; สิ่งที่คุณส่ง (หรือไม่ส่ง)
จะเป็นตัวกำหนด **ว่าใครควบคุม TG ของคุณ**:

| hotspot ของคุณส่ง | ใครเป็นผู้กำหนด TG ของคุณ | สิ่งที่เกิดขึ้น |
|---|---|---|
| `OPTIONS=PASS=รหัส_ของคุณ;` | **บริการตนเอง** (แผงเว็บ) | เซิร์ฟเวอร์จะตรวจสอบรหัสผ่านของคุณแล้วอ่าน TG จากฐานข้อมูล **คุณสามารถล็อกอินเข้า dashboard ด้วยรหัสผ่านและตาม IP ได้** |
| `OPTIONS=` ว่าง | **บริการตนเอง** (แผงเว็บ) | เซิร์ฟเวอร์จะอ่าน TG จากฐานข้อมูล คุณสามารถใช้ **การล็อกอินอัตโนมัติตาม IP** บน dashboard ได้เท่านั้น (ไม่มีรหัสผ่าน) |
| ไม่มี OPTIONS (ผ่านไป 10 วินาที) | **บริการตนเอง** (แผงเว็บ) | เซิร์ฟเวอร์จะถือว่า hotspot ของคุณไม่มี OPTIONS ของตัวเองและใช้ฐานข้อมูล เหมือนกรณีว่าง |
| `OPTIONS=TS2=730;` (พร้อม TG, SINGLE ฯลฯ) | **hotspot ของคุณ** | เซิร์ฟเวอร์จะรับ TG จากบรรทัดโดยตรง **ไม่สนใจแผงเว็บ** คุณสามารถใช้ **การล็อกอินอัตโนมัติตาม IP** บน dashboard ได้เท่านั้น (ไม่มีรหัสผ่าน) |

> **สำคัญ:** หาก hotspot ของคุณ **ไม่** ส่ง `PASS=` **คุณจะไม่สามารถ
> ล็อกอินเข้า dashboard ด้วยรหัสผ่านได้** คุณสามารถใช้การล็อกอินอัตโนมัติตาม IP เท่านั้น
> (หาก IP ของคุณตรงกัน) หากต้องการล็อกอินด้วยรหัสผ่าน hotspot ของคุณ
> ต้องส่ง `OPTIONS=PASS=รหัส_ของคุณ;` ในการตั้งค่า (Pi-Star / WPSD:
> ฟิลด์ `optsfile`) รหัสผ่านต้องตรงกับที่คุณลงทะเบียนไว้ใน
> แผง

### TG แบบสแตติก (เปิดอยู่เสมอ)

TG ที่คุณใส่ในการตั้งค่าหรือแผงจะคงอยู่ **ในโหมดฟังเสมอ** โดยที่
คุณไม่ต้องทำอะไรเลย ตัวอย่าง: หากคุณตั้งค่า `TS2=730` hotspot ของคุณจะ
ฟัง 730 อย่างถาวรจนกว่าคุณจะลบออก

### TG แบบไดนามิก (คุณเปิดใช้งานเอง)

TG ที่คุณเปิดใช้งานด้วย PTT (โดยไม่ได้ตั้งค่าไว้) เป็น **แบบไดนามิก**:
จะอยู่ได้ตามเวลาที่กำหนด (ที่ผู้ดูแลระบบตั้งไว้ มักจะประมาณ 10 นาที)
แล้วจะล้างตัวเอง หรือถูกล้างเมื่อคุณกด 4000

### โหมด SINGLE (ฟังเฉพาะ)

hotspot บางรุ่นมี **SINGLE=1** ซึ่งหมายความว่า: **มีเพียง TG ไดนามิกเดียวต่อ slot ในแต่ละช่วงเวลา**
หากคุณเปิดใช้งาน TG ใหม่ มันจะแทนที่ TG ก่อนหน้า

รุ่นอื่น ๆ มี **SINGLE=0** ซึ่งอนุญาตให้ TG ไดนามิกหลายตัวสะสมได้ ผู้ดูแลระบบของคุณ
จะเป็นผู้ตัดสินใจว่าจะใช้แบบใด

## ปัญหาที่พบบ่อยและวิธีจัดการ

| ปัญหา | สาเหตุที่เป็นไปได้ | สิ่งที่ควรทำ |
|---|---|---|
| **ฉันไม่ได้ยินอะไรเลย** | TG ของคุณไม่ได้เปิดใช้งาน หรือ slot ไม่ว่างด้วย TG อื่น | ส่งสัญญาณบน TG นั้นเพื่อเปิดใช้งาน; หรือรอให้ QSO อื่นจบ |
| **เสียงของฉันไม่ผ่าน** | TG ไม่ว่างเพราะมีคนอื่นอยู่ | รอให้การสนทนาสิ้นสุดแล้วลองอีกครั้ง |
| **เสียงถูกตัดตอนต้น** | slot อยู่ในช่วง hangtime | เป็นเรื่องปกติ; รอ 5 วินาทีแล้วทำซ้ำ |
| **ฉันเปิดใช้งาน TG แต่ไม่ได้ยิน** | ไม่มีใครส่งสัญญาณบน TG นั้นในขณะนี้ | TG ถูกเปิดใช้งานแล้วแต่เงียบ; เมื่อมีคนพูด คุณจะได้ยิน |
| **TG ไดนามิกของฉันหายไป** | หมดเวลาหรือมีคนกด 4000 | ส่งสัญญาณบน TG นั้นอีกครั้งเพื่อเปิดใช้งานใหม่ |
| **ฉันพูดไม่ได้ ถูกละเลย** | มีคนอื่นกำลังใช้ TG นั้นอยู่ | รอให้ถึงคิวของคุณ (ฟังก่อน) |

1. **ฟังก่อนพูด** ก่อนกด PTT รอสักครู่เพื่อดูว่า
   มีคนอื่นกำลังพูดอยู่หรือไม่ กฎ “หนึ่งการสนทนาต่อ TG” จะช่วยไม่ให้คุณ
   แทรกขัดจังหวะ แต่การฟังก่อนก็เป็นมารยาทที่ดี

2. **แจ้งตัวตน** กล่าว callsign ของคุณที่ต้นและที่ปลาย เป็น
   ข้อกำหนดตามกฎระเบียบและช่วยให้ผู้อื่นทราบว่าใครกำลังพูด

3. **หยุดพักระหว่างตา** เว้น 2-3 วินาทีระหว่างข้อความของคุณกับ
   ของอีกฝ่าย ทำให้ผู้อื่นมีเวลาเข้าร่วมสนทนาและหลีกเลี่ยง
   การถูกตัด

4. **อย่ากด PTT เร็วเกินไป** หากคุณปล่อยและกดอีกครั้งในเวลาน้อย
   กว่าหนึ่งวินาที เซิร์ฟเวอร์อาจประมวลผลการเปลี่ยนแปลงไม่ถูกต้อง การหยุดสักครู่
   จึงดีต่อสุขภาพ

5. **หากคุณเปิดใช้งาน TG ไดนามิก ให้กด 4000 เมื่อเสร็จ** จะช่วย
   ปลดปล่อยทรัพยากรและหลีกเลี่ยงการได้ยินการจราจรที่คุณไม่สนใจอีกต่อไป
